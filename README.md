# Aircraft-Aeroelastic-Control-Benchmark

**This repository is _under construction_** 

This comprehensive benchmark allows to couple a MATLAB
control framework to the SHARPy nonlinear aeroelastic
solver, effectively creating a virtual hardware-in-the-loop environment.

Sharpy Aeroelastic Toolbox: 
https://github.com/ImperialCollegeLondon/sharpy

## Implementation Guidlines:
This benchmark utilizes structural and aerodynamic H5 files generated by the SHARPy Aeroelastic toolbox. As such, please refer to https://ic-sharpy.readthedocs.io/en/latest/ for installation instructions. SHARPy is a Linux based open-source aeroelastic toolbox. 

Benchmark files necessary for MatlabFlex scripts are generated in Jupyter file Benchmark.ipynb. Originally, it was required to import a MATLAB engine API within the Linux subsystem; the benchmark circumvents this with the MATLAB script Run_Benchmark_Test.m and commenting out any engine mentions in the Jupyter file.

The following is a set of instruction that may help run the Benchmark Cases:
1. Setup Linux Distribution and Python Environments.
   - Benchmark has been created and tested in WSL Ubuntu, so it is recommended to do the same.
   - Download a python packager such as miniconda or Anaconda3 to use the (sharpy) environment. 
     
2. Install SHAPRy aeroelastic toolbox
   - Preferable locations is directly in the home/user/ folder.
   - SHARPy depends on building Mayavi, which has recently has issues building with upgrades in Python, VTK, and Numpy. Please see troubleshooting for recommondations on how to get around these issues.
   - Ensure that the (sharpy) environment is active when attempting to run the Benchmark.
   
3. Clone Repository
   - Recommended to clone repository using git clone to the same home/user/ folder which SHARPy has been cloned to.
   - May have to run sudo apt install gh prior.

4. Accessing and Generating Build Scripts
   - For best results it is recommended to install VS_Code with the "Remote Development" Extension.
   - CD to the Benchmark directory in Linux. Use "code . " to accesses the Benchmark.ipynb file.
   - In the top right select the sharpy Kernel.
   - If you have a MATLAB engine API installed on the Linux distro run the Benchmark. For running fully coupled aircraft cases ensure to change any file locations to the apporpriate address.
   - If you do not have MATLAB engine API, whenever the engine is called switch over to Run_Benchmark_Test.m and change the run settings as needed to depending on the case.

## MATLAB Repository structure:
The current Bencmark structure is setup for the Pazy Wing FEM. Flight parameters such as angle of attack, airspeed, throttle, 
###    /TestBenchPazy: Directory contatining files generated by SHARPy and Benchmark scripts. Simulation results and run packages are found here for case simulations and post processesing. This folder acts as an interface between SHARPy and the benchmark.  
###    /MatlabFlex: Benchmark project scripts and functions.
####   ~/configs/sim_init.m:
####   ~ sim_run.m
####   ~/rom: structural and aerodynamic ROM data; modal maps; gravity and follower-force mappings.
####    ~/plant: time-stepper, outputs, sensor models.
####    ~/est: MHE problem; arrival covariance; window management; warm starts.
####    ~/ctrl: MPC multiple shooting; constraints; warm starts; LQR design.
####    ~/fusion: complementary filters and interface to actuator chain.
####    ~/act: servo model, saturation, rate limits, anti-windup.
 ####   ~/cases: A/B/C scripts with identical seeds and logging.
 ####   ~/post: metrics (GLA\%, RMS), actuator statistics, timing tables, plots.
 ####   ~/validate: high-fidelity cross-check scripts and plotting templates.
